## 2024-05-11 - [Path Traversal in API Response Parsing]
**Vulnerability:** The application unsafely constructs file paths using `os.path.join` with the `FileName` field fetched directly from external API responses. If an attacker controls the API response or intercepts it to return an absolute path (e.g. `/etc/passwd`) or relative traversal payload (`../../`), it causes `os.path.join` to overwrite arbitrary system files when the image download is written.
**Learning:** `os.path.join` on Linux will completely replace the base path with the incoming path if the incoming path starts with a `/`. It also natively supports relative path traversal like `../`. We must never trust any input—even API payloads from trusted third-parties like SmugMug—that dictates file paths without thorough sanitization.
**Prevention:** Always use `os.path.basename` (along with standardizing path separators via `.replace('\\', '/')` to prevent Windows-style path traversal tricks escaping Linux validation) on any externally sourced filenames before joining them to output directories.
